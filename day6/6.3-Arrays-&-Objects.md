# **6.3 JavaScript Arrays & Objects (Job-Focused Guide)**

Arrays and objects are **core data structures in JavaScript**.
In real projects, they are used to **store, manipulate, and render data dynamically**.

üíº **In jobs**, knowledge of arrays & objects is essential for:

* API response handling
* React state management
* Loops and conditional rendering
* Dynamic UI components

---

## **1Ô∏è‚É£ Arrays**

Arrays are **ordered lists of values**.

### **Declaration**

```javascript
const numbers = [1, 2, 3, 4];
let fruits = ["Apple", "Banana", "Orange"];
```

### **Real-World Use Cases**

* Storing user data from APIs
* Lists of items in a shopping cart
* Rendering menus, tables, or cards dynamically

### **Common Array Methods**

| Method      | Example                                | Use Case                        |
| ----------- | -------------------------------------- | ------------------------------- |
| `push()`    | `fruits.push("Mango")`                 | Add item at end                 |
| `pop()`     | `fruits.pop()`                         | Remove last item                |
| `shift()`   | `fruits.shift()`                       | Remove first item               |
| `unshift()` | `fruits.unshift("Kiwi")`               | Add item at beginning           |
| `map()`     | `fruits.map(f => f.toUpperCase())`     | Transform arrays (UI rendering) |
| `filter()`  | `numbers.filter(n => n > 2)`           | Filter API data                 |
| `find()`    | `fruits.find(f => f === "Banana")`     | Search specific value           |
| `forEach()` | `numbers.forEach(n => console.log(n))` | Loop through arrays             |

---

### **Real-World Example**

```javascript
const users = [
  { id: 1, name: "Alice", role: "admin" },
  { id: 2, name: "Bob", role: "user" },
];

users.map(user => <div key={user.id}>{user.name}</div>);
```

üíº **Used in React** to render lists of items dynamically.

## **2Ô∏è‚É£ Objects**

Objects store **key-value pairs**.

### **Declaration**

```javascript
const user = {
  id: 1,
  name: "Alice",
  email: "alice@example.com",
  isAdmin: true
};
```

### **Real-World Use Cases**

* API responses (`user`, `post`, `product`)
* Configuration settings
* Component props in React

### **Accessing Properties**

```javascript
console.log(user.name); // Alice
console.log(user['email']); // alice@example.com
```

### **Modifying Properties**

```javascript
user.isAdmin = false;
user.age = 22; // Add new property
```

---

## **3Ô∏è‚É£ Arrays of Objects (Very Common in Jobs)**

Most API responses return **arrays of objects**:

```javascript
const products = [
  { id: 1, name: "Laptop", price: 500 },
  { id: 2, name: "Phone", price: 300 },
];

// Filter products above $400
const expensive = products.filter(p => p.price > 400);

// Map to JSX in React
products.map(p => <div key={p.id}>{p.name}: ${p.price}</div>);
```

üíº This is **exactly how data is rendered in dashboards, e-commerce, or SaaS apps**.

## **4Ô∏è‚É£ Best Practices (Industry-Level)**

‚úÖ Use **const** for arrays/objects unless reassignment needed
‚úÖ Prefer **map/filter/find** over manual loops for UI rendering
‚úÖ Keep objects structured and consistent (same keys)
‚úÖ Avoid deeply nested objects if possible; normalize API data
‚úÖ Destructure objects in functions/components for clarity
