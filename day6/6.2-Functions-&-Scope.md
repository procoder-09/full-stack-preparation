# **6.2 JavaScript Functions & Scope (Job-Focused Guide)**

Functions are **reusable blocks of code** that perform a specific task.
Scope determines **where variables are accessible**.

üíº **In real jobs**, functions and scope are essential for:

* Organizing code in React/Node apps
* Handling API responses
* Event handling
* State management
* Writing maintainable, modular code

## **1Ô∏è‚É£ Functions in JavaScript**

### **Types of Functions**

| Type                       | Syntax                       | Real-World Use Case                          |
| -------------------------- | ---------------------------- | -------------------------------------------- |
| Function Declaration       | `function greet(){}`         | Reusable logic, API calls, utility functions |
| Function Expression        | `const greet = function(){}` | Pass functions as arguments                  |
| Arrow Function             | `const greet = () => {}`     | React event handlers, short functions        |
| IIFE (Immediately Invoked) | `(function(){})()`           | One-time initialization (rare in React)      |

## **2Ô∏è‚É£ Function Scope**

### **1. Global Scope**

* Accessible anywhere in the file
* Avoid polluting global scope in real projects

```javascript
let siteName = "MyApp"; // Global
function showName() {
  console.log(siteName); // Accessible
}
```

### **2. Local/Function Scope**

* Variables declared inside a function are **not accessible outside**

```javascript
function greet() {
  let message = "Hello"; // Local
  console.log(message);
}
console.log(message); // ‚ùå Error
```

### **3. Block Scope**

* Variables declared with `let` or `const` inside `{}` are **block scoped**

```javascript
if(true) {
  let age = 22;
  const name = "Ramya";
}
console.log(age); // ‚ùå Error
```

üíº **Why scope matters in jobs**

* Prevents variable conflicts
* Ensures proper state management in React
* Reduces bugs in backend Node.js code

## **3Ô∏è‚É£ Closures (Intermediate, Real-World)**

A closure is a function that **remembers the scope in which it was created**.

```javascript
function counter() {
  let count = 0;
  return function() {
    count++;
    return count;
  };
}

const increment = counter();
console.log(increment()); // 1
console.log(increment()); // 2
```

üíº **Used in production**

* State management in vanilla JS
* React hooks like `useState` internally use closures
* Utility functions with private variables

## **6Ô∏è‚É£ Real-World Example: React Event + Function + Scope**

```javascript
function TodoApp() {
  const [tasks, setTasks] = React.useState([]);

  const addTask = (task) => {
    setTasks([...tasks, task]); // Local state scope
  };

  return (
    <button onClick={() => addTask("New Task")}>
      Add Task
    </button>
  );
}
```

* `addTask` is a **function managing local state**
* `tasks` is **block scoped to the component**
* Very typical in **React projects**
